Professional Self-Assessment

     My computer science degree has taught me a lot of things that will help my professional career. I’ve definitely learned a lot more than I thought I would. Before going into computer science, I really had no idea what I wanted to do. I started working as an IT assistant and I really do enjoy what I was doing. I liked working with computers and I felt like I was pretty decent at it. I decided on computer science because I wanted a degree that could get me various types of jobs in case I decided I didn’t want to go into IT. The courses that required programming where a lot of fun and something I would definitely like to do as a career. I really hadn’t done a lot of programming before so I really didn’t know what to expect in those classes. I enjoyed the problem solving and it felt really good to complete a task assignment. I started to care about why my assignments worked they way they did or often times why what I was trying didn’t work. Many classes such as collaboration and team projects, software development life cycle, software testing, and data structures & algorithms are classes that I think will help me for the rest of my programming career.

     My class on Agile software development life cycle taught me how developing software works and how to communicate with other members on a team. Agile shows the benefit of working as a team to develop multiple, smaller iterations of a project as opposed to management handing a project down the line until the last team, creating the final project. It taught the importance of keeping communication with everyone, from developers and testers to product owner. The class also showed that there are many other methodologies besides agile that can be used. Some projects may not work with the agile method and that’s okay. Another class was dedicated to data structures and algorithms. I really enjoyed this class because it was difficult and really showed that sometimes, things were a lot more simple than I tried to make them. Learning the various data structures like stacks, linked lists, trees, and of course arrays seem like something that will be invaluable for my entire developer career. Learning various algorithms showed me that there are multiple ways to do one thing. Some algorithms are better for inserting but at the same time might not be good at deleting. Others might be good at searching in a massive data structure but very poor in a small structure. Another class was object oriented design and analysis which showed me that there are many steps that can be done prior to developing that will make developing significantly easier. Making UML diagrams like various behavioral and structural diagrams. This allows you to understand how to code a project since things like classes and use cases are all thought about how they relate to each other. This means you just have to do the actually coding. The last class that I think will help me greatly was about secure coding. The biggest thing I learned from this was to follow best coding practices at all times and to take code reviews seriously. Bugs that were in the real world could have been easily caught if they were properly reviewed and some would have made a bug irrelevant if they just kept to coding standards. A lot of the mistakes that people make are either from lazily coding or code reviews not being serious. The class also introduced me to static analysis tools. This can search your code for common bugs that occur with the programming language you use such as integer overflows, string buffer overflows, or null pointers. This would be more difficult for someone to catch without specifically testing for it.

     My project is a good showcase for my skills for my ePortfolio because it shows that I have an understanding on various subjects within programming. It shows my ability to use software design with making an activity diagram and a use case diagram. This made making my bank program a lot easier because it gave me a basic understand of how I wanted to code my program. I could even code sections of my program while I researched how to make other parts of my program work. Getting hung up on part of my program didn’t completely stop me from programming the classes I did know how. It also shows that my knowledge of algorithms and how to code them for things beyond regular arrays. I used the normal algorithms and changed them so I could sort by various variables in an object array. This shows that my knowledge of algorithms is good enough to alter them depending on how I need to use them. My program also shows that I can communicate with a database. I used Microsoft Access which may not be a popular way to organize a database but it shows a proof of concept that I can work with what I have. After a lot of research, I was able to communicate with the database even after java and MS Access had gone through major updates changing how they can communicate with each other. I had a big problem of communicating and was holding up a big portion of my project but I was able to persevere and was finally able to get everything to work. I feel like I’ve learned a lot and I know that I have even more to learn.
